# Code of Conduct

# SISDIGEM Document Studio

## Our Commitment

SISDIGEM Document Studio is an Open Source initiative created to build practical software that solves real-world problems.

We are committed to fostering an open, respectful, inclusive, and collaborative environment where developers, designers, researchers, students, and users from around the world can contribute regardless of experience level.

Our goal is simple:

> Build software that saves people time.

---

# Our Values

We encourage every community member to:

- Be respectful.
- Share knowledge.
- Help beginners.
- Accept constructive feedback.
- Collaborate professionally.
- Focus on solving problems.
- Respect different opinions and cultures.

Every contribution, regardless of size, is valuable.

---

# Expected Behavior

Community members are expected to:

- Use respectful language.
- Be patient with new contributors.
- Report bugs responsibly.
- Provide constructive suggestions.
- Keep discussions technical and productive.
- Respect project decisions.
- Credit the work of others.

---

# Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or discrimination.
- Personal attacks.
- Offensive language.
- Spam.
- Publishing malicious code.
- Intentionally breaking the project.
- Sharing illegal content.
- Violating intellectual property rights.

Contributors engaging in these behaviors may have their contributions removed or their participation restricted.

---

# Project Philosophy

This project was born from a simple observation:

Many professionals lose valuable hours every week performing repetitive document conversion tasks.

Instead of accepting that reality, we decided to build tools that automate those processes.

Our philosophy is:

> Software should eliminate repetitive work so people can focus on creating value.

Every new feature should move the project closer to that mission.

---

# Community Guidelines

When contributing:

- Keep pull requests focused.
- Write clean and understandable code.
- Document new features.
- Preserve backward compatibility whenever possible.
- Test your changes before submitting.
- Discuss major changes before implementation.

Small, well-designed contributions are preferred over large undocumented changes.

---

# Open Source Principles

This project welcomes contributions including:

- Bug fixes
- New document formats
- Performance improvements
- User interface enhancements
- AI integrations
- Documentation improvements
- New language translations
- Automation features
- Testing
- Examples

Whether you contribute one line of code or an entire module, your work is appreciated.

---

# Enforcement

Project maintainers are responsible for maintaining a welcoming environment.

They may edit, remove, or reject comments, commits, pull requests, issues, or contributions that violate this Code of Conduct.

---

# Reporting Issues

If you experience unacceptable behavior or discover conduct that threatens the health of the community, please open a private issue or contact the maintainers.

All reports will be reviewed fairly and confidentially whenever possible.

---

# Our Vision

SISDIGEM is more than a document converter.

It is an engineering platform dedicated to building software that solves everyday problems through:

- Automation
- Artificial Intelligence
- Document Engineering
- Data Processing
- Open Source Collaboration

We believe software should be accessible, transparent, practical, and useful for everyone.
