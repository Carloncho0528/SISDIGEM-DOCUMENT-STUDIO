# Contributing to SISDIGEM Document Studio

First of all, thank you for your interest in contributing to SISDIGEM Document Studio.

This project exists because real-world problems deserve simple, reliable and open solutions.

Whether you are fixing a typo, adding support for a new document format, improving performance or creating a completely new module, your contribution is welcome.

---

# Our Mission

Build professional Open Source tools that help people automate repetitive document processing tasks.

Every contribution should support one or more of these goals:

- Simplicity
- Performance
- Reliability
- Scalability
- Accessibility

---

# Ways You Can Contribute

## Bug Reports

Open an Issue describing:

- Operating System
- Python version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if available

---

## Feature Requests

Explain:

- The problem
- Why it matters
- Your proposed solution
- Alternative solutions

---

## Pull Requests

Before submitting:

- Keep the code clean
- Comment important logic
- Follow project structure
- Test your changes
- Update documentation if necessary

---

# Coding Standards

Python

- PEP8
- Meaningful variable names
- Modular code
- Small functions

C++

- Modern C++17+
- RAII
- STL whenever possible
- Clear architecture

---

# Branch Strategy

main

Stable production version

develop

Next version under development

feature/

New features

bugfix/

Bug fixes

---

# Commit Messages

Examples:

Add PDF to DOCX converter

Improve HTML parser

Fix CSV export bug

Refactor conversion engine

Update README
