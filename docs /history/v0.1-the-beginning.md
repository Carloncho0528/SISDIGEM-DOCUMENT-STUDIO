# Version 0.1 — The Beginning

**Release:** Version 0.1  
**Language:** Python  
**Status:** Initial Prototype  
**Date:** 2026

---

# The Beginning

Every software project starts with a question.

For SISDIGEM Document Studio, that question was simple:

> **How can we process a large amount of information without spending hours doing repetitive manual work?**

This project was not created as a document converter.

It was born while solving a real-world engineering problem.

---

# The Problem

During several research projects, large collections of HTML documents needed to be analyzed.

Opening each file individually, copying the content, cleaning the text, and preparing it for analysis required a significant amount of time.

The process looked like this:

- Open an HTML file.
- Copy the content.
- Clean unnecessary information.
- Paste it into another application.
- Repeat the process again.
- Repeat dozens or even hundreds of times.

Although each step was simple, together they consumed valuable time and increased the risk of human error.

---

# The First Idea

Instead of repeating the same task manually, a question emerged:

> **Why not let software perform this repetitive work automatically?**

The goal was modest:

Read an HTML file.

Extract the useful information.

Export it into a structured format that could be analyzed later.

No graphical interface.

No multiple formats.

No automation.

Just solving one problem efficiently.

---

# The First Prototype

The first prototype was intentionally simple.

It was developed in Python using only a few libraries.

Main technologies:

- BeautifulSoup
- Pandas

The workflow was straightforward:

HTML File

↓

Read Document

↓

Extract Messages

↓

Create DataFrame

↓

Export CSV

This small script represented the foundation of everything that would come later.

---

# Version 0.1 Features

- HTML parsing
- Message extraction
- User extraction
- Date and time extraction
- CSV export
- UTF-8 support
- Simple and readable code

---

# First Source Code

The first implementation consisted of a single Python script.

Its purpose was to demonstrate that repetitive document processing could be automated with only a few lines of code.

Although simple, it successfully transformed raw HTML data into structured information ready for further analysis.

---

# Results

The prototype achieved its primary objective.

Instead of manually processing documents one by one, the entire extraction process could be completed automatically in a matter of seconds.

The project demonstrated that:

- Repetitive work could be automated.
- Human error could be reduced.
- Structured datasets could be generated consistently.
- The solution was scalable.

---

# Lessons Learned

Version 0.1 taught an important lesson.

The real challenge was not converting HTML files.

The real challenge was eliminating repetitive manual work.

This realization completely changed the direction of the project.

Instead of building another HTML parser, the vision expanded toward creating a universal document processing platform.

---

# Limitations

Although successful, Version 0.1 had several limitations.

It could only:

- Process one HTML file at a time.
- Export to CSV only.
- Operate from the command line.
- Handle a specific HTML structure.

It did not include:

- Graphical user interface.
- Batch processing.
- Excel export.
- Error logging.
- Folder processing.
- Support for additional document formats.

These limitations became the roadmap for the next version.

---

# Why Version 0.2 Was Necessary

As the number of documents increased, processing one file at a time was no longer practical.

Users needed a solution capable of processing complete folders automatically.

This requirement marked the beginning of Version 0.2.

The project evolved from a simple script into the first desktop application.

---

# Engineering Philosophy

This first version established a principle that continues to guide the project today.

> **Software should eliminate repetitive work so people can focus on thinking, creating, and solving problems.**

Every future version of SISDIGEM Document Studio is built around this idea.

---

# Historical Importance

Version 0.1 represents the birth of SISDIGEM Document Studio.

Although technically simple, it laid the foundation for a platform whose long-term vision extends far beyond document conversion.

It marked the beginning of an engineering journey focused on automation, intelligent document processing, and Open Source collaboration.
